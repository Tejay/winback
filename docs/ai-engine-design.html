📝 Changelog Update
Founder posts what they've shipped
✓ Working (keyword extraction)
File: app/api/changelog/route.ts
What it does: LLM extracts 3–8 keywords from the changelog text. Then SQL ILIKE matches those keywords against each subscriber's triggerKeyword.
Weakness: Substring match only. "CSV export" won't match "spreadsheet improvements." Semantic matching (embeddings or LLM re-rank) would catch these.
🕐 Time Decay
90-day backstop re-engagement for passive churners
✓ Working (spec 16)
File: app/api/cron/reengagement/route.ts
How it works: Fixed 90-day backstop (not LLM-picked). Daily cron at 09:00 UTC finds subscribers where cancelledAt + 90 days ≤ now() and status is 'contacted'.
What it does:
  • Re-classifies with fresh context (current changelog, elapsed time)
  • If AI says contact → sends re-engagement email via Resend
  • If AI says suppress → marks done, no email
  • Max 1 re-engagement per subscriber
  • Skips if changelog already triggered a win-back email
Design choice: Single fixed interval, not LLM-driven 30/90/180. We can't predict passive subscriber receptivity — this is a "one last try" backstop.

🎯 Classification
Tier 1–4 + reason + confidence + category
✓ Working
File: src/winback/lib/classifier.ts (system prompt, lines 46–72)
Output:
  • tier: 1 (targeted), 2 (ask for detail), 3 (generic re-engage), 4 (suppress)
  • cancellationReason: short phrase for dashboard
  • cancellationCategory: Competitor | Price | Quality | Unused | Feature | Other

Model: claude-haiku-4-5-20251001, temp 0, max 1500 tokens. Zero-data-retention is an org-level setting (the deprecated anthropic-beta: zero-retention header was removed).

💬 Send Reply Response
When subscriber replies, send a follow-up that addresses what they said
✓ Fixed (spec 15) — threaded follow-up with guardrails
What happens now:
  • Reply arrives → re-classifies with reply text → new firstMessage generated
  • sendReplyEmail() sends it as a threaded follow-up using In-Reply-To / References headers
  • Respects DNC + customer-paused + Tier 4 suppress checks
  • Recorded as type 'followup' in emailsSent
  • Reactivation link + unsubscribe link appended to every follow-up
Guardrails:
  • Max 2 follow-ups per subscriber (MAX_FOLLOWUPS = 2). After that, AI stops replying.
  • When the limit is hit, the founder is notified by email with the subscriber's details and latest reply — "this one needs your personal touch."

Status: Missing. The classifier prompt is static. It doesn't know that "price-cited cancellations with tenure>180d recover at 12% with an offer and 3% without." Feeding cohort stats into the system prompt is the unlock for genuine AI learning.
